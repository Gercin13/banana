# Nano Studio

Локальный генератор изображений на Google Gemini (**Nano Banana**). Максимально простой интерфейс: промпт (с голосовым вводом), все популярные соотношения сторон, 1–4 изображения за раз. Спроектирован так, чтобы позже легко открыть его для других пользователей и сделать безопасным.

## Ключ API
Google AI Studio → **Get API key** (https://aistudio.google.com). Локально кладётся в `.env` как `GEMINI_API_KEY=...`; на Netlify — в переменные окружения сайта.

## Локальный запуск
Вариант A — Express:
```bash
npm install
cp .env.example .env      # впишите GEMINI_API_KEY
npm start                  # → http://localhost:3000
```
Вариант B — эмуляция Netlify (функции + редиректы `/api/*`):
```bash
npm install
cp .env.example .env
npx netlify dev            # → http://localhost:8888
```

## Деплой на Netlify
1. Залейте репозиторий на GitHub и подключите в Netlify (**Add new site → Import from Git**).
2. Настройки сборки подхватятся из `netlify.toml` (publish `public`, functions `netlify/functions`); build command можно оставить пустым.
3. **Site settings → Environment variables** → добавьте `GEMINI_API_KEY`.
4. Deploy. Фронтенд отдаётся статикой, `/api/*` идёт в функции (редирект в `netlify.toml`).

**Лимит времени функций:** синхронная функция ~10 сек (free) / до 26 сек (paid). «Черновик» (Nano Banana 2) укладывается; «Качество» (Pro) медленнее — при таймаутах уменьшите количество изображений или возьмите план с лимитом 26 сек.

**Лимит размера ответа (важно для 2K/4K):** ответ функции Netlify ограничен ~6 МБ. Изображения возвращаются как base64 в JSON, поэтому одно **4K**-изображение может превысить лимит, а несколько **2K** — тем более. На Netlify для 4K держите количество = 1; надёжное решение для высоких разрешений — сохранять изображения в хранилище (S3/R2) и возвращать ссылки вместо base64 (следующий шаг развития). Локально (Express) таких лимитов нет.

## Возможности (MVP)
- **Промпт с опциональным голосовым вводом** (Web Speech API — Chrome/Edge; если браузер не поддерживает, кнопка микрофона скрывается).
- **Все популярные соотношения сторон:** 1:1, 4:5, 5:4, 3:4, 4:3, 2:3, 3:2, 9:16, 16:9, 21:9.
- **1–4 изображения** за одну генерацию (выполняются параллельно).
- **Два режима качества:** Черновик (Nano Banana 2 — быстро/дёшево) и Качество (Nano Banana Pro).
- **Референсы для консистентного персонажа:** две зоны — «Лицо / личность» и «Остальное» (позы, одежда, стиль), суммарно до 14. Лицо отправляется первым и мягко «якорит» личность (backend добавляет короткую инструкцию идентичности). Референсы уменьшаются до 1024px на клиенте перед отправкой.
- **Разрешение 1K / 2K / 4K** (для «Черновик» и «Качество»).
- Галерея результатов со скачиванием.
- `Ctrl / ⌘ + Enter` — быстрый запуск генерации.

## Структура
```
nano-studio/
├── public/                 статичный фронтенд (vanilla JS, без сборки)
│   ├── index.html
│   ├── styles.css
│   └── app.js
├── lib/
│   ├── engine.js           GenerationEngine — абстракция над бэкендом генерации
│   └── handler.js          общая логика (валидация + оркестрация), DRY
├── netlify/functions/      serverless API (продакшн на Netlify)
│   ├── generate.mjs
│   └── health.mjs
├── server.js               локальный Express-сервер (опционально, npm start)
├── netlify.toml            конфиг Netlify (публикация, функции, редирект /api/*)
├── .env.example
└── package.json
```
Ключ провайдера живёт только на сервере (Express `.env`) или в env-переменной Netlify-функции — браузер его не видит. Express и Netlify-функции используют **одну и ту же** логику из `lib/handler.js`.

## Архитектура и рост до мультипользователя
Ключ провайдера живёт **только** на backend; браузер его не видит. Переход «для других» — это набор изолированных изменений, а не переписывание:

| Слой | Сейчас (локально) | Потом (мультипользователь) | Где менять |
|---|---|---|---|
| Аутентификация | нет (один юзер) | сессии / OAuth / JWT | `getUserId()` + `lib/handler.js` (Express и функции) |
| Хранение | не сохраняется (возврат в браузер) | БД + объектное хранилище (S3/R2), ключ по `userId` | «шов» в обработчике `/api/generate` |
| Квоты/лимиты | нет | на пользователя | `/api/generate` |
| Движок | Gemini Nano Banana | FLUX / локальный ComfyUI / др. | `lib/engine.js` |
| Референсы | реализованы: 2 зоны (лицо + остальное), до 14 | доп. слоты (отдельно одежда), пресеты образов | UI + `refs` в движке |

## Безопасность и приватность
- Ключ — только на сервере (`.env`), никогда во фронтенде.
- Валидация входа (промпт, соотношение сторон, количество, режим), CORS same-origin.
- Если позже будете загружать лица реальных людей (референсы) — нужно **согласие**; не использовать для генерации без разрешения.
